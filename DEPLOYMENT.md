# Deploy TrueSEO on Vercel

## 1. Push the repo

Connect this Git repository to [Vercel](https://vercel.com) (Import Project → pick the repo → Framework Preset: Next.js).

## 2. Environment variables (Production)

In **Vercel → Project → Settings → Environment Variables**, add at least:

| Variable | Required? | Purpose |
|----------|-----------|---------|
| `NEXT_PUBLIC_SITE_URL` | **Yes (production)** | Canonical base URL, e.g. `https://www.yoursite.com`. Used by `sitemap.xml`, `robots.txt`, and `metadataBase` in `src/app/layout.js`. Without it on Vercel, preview URLs use `VERCEL_URL` automatically. |
| `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION` | No | Content value from Google Search Console HTML-tag verification. |
| `NEXT_PUBLIC_BING_SITE_VERIFICATION` | No | Bing Webmaster HTML meta content. |
| `NEXT_PUBLIC_YANDEX_VERIFICATION` | No | Yandex verification string. |
| `NEXT_PUBLIC_ADSENSE_CLIENT` | No | e.g. `ca-pub-xxxxxxxxxxxxxxxx` — loads the AdSense script when set. |
| `NEXT_PUBLIC_ADSENSE_ADS_TXT_LINE` | No | One full line for `ads.txt` (see AdSense site settings). If unset, `/ads.txt` returns 404. |

Copy from `.env.example` and adjust values.

## 3. Google Search Console

1. Add your **exact production URL** as a property (URL-prefix or Domain).
2. Verify ownership using the **HTML tag** method and paste the token into `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION` in Vercel, then redeploy.
3. After deploy, open **Sitemaps** and submit:  
   `https://YOUR_DOMAIN/sitemap.xml`  
   (This file is generated by `src/app/sitemap.js` and lists `/`, `/about`, `/contact`, `/privacy`, `/terms`, and every `/tools/...` page.)
4. Use **URL Inspection** on the home page and a few tool URLs to request indexing if you want faster discovery.

## 4. Google AdSense

1. In AdSense, add your site and follow the verification steps.
2. Set `NEXT_PUBLIC_ADSENSE_CLIENT` to your publisher ID (`ca-pub-...`). The root layout injects the official script when this is set.
3. Set `NEXT_PUBLIC_ADSENSE_ADS_TXT_LINE` to the exact line AdSense shows for `ads.txt` (e.g. `google.com, pub-..., DIRECT, f08c47fec0942fa0`). The route `src/app/ads.txt/route.js` serves it at `/ads.txt`.
4. Redeploy after changing env vars.

## 6. Local build check

```bash
npm install
npm run build
```

Fix any build errors before relying on Vercel’s build.

## 7. Notes

- **Serverless limits**: WHOIS, raw TLS, and some heavy probes may hit Vercel’s function timeout on slow hosts. Increase **max duration** in `vercel.json` or the dashboard if needed.
